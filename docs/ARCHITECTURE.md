# FOCUS.bm — архитектура жүйесі

## 1. Жалпы тәсіл: модульдік монолит → микросервистер

**MVP және ерте саты:** **модульдік монолит** (бір код базасы, логикалық модульдер: auth, learning, messaging, social, dating, admin).

**Неге алдымен монолит:**
- Жылдам итерация, бір деплой, қарапайым транзакциялар (PostgreSQL).
- Команда кіші болса, микросервистердің операциялық құны (tracing, контракттар, CI/CD) артық.

**Миллиондарға масштабтау кезінде:** домендерді бөлу — **микросервистер немесе «модульдік монолит + оқиға шина»**:
- **Auth & Identity** — JWT/OAuth2, сессиялар, құрылғылар.
- **Learning** — курстар, медиа метадеректер, прогресс.
- **Messaging** — тек шифрланған конверттерді сақтау/ретрансляция, WebSocket шлюзі.
- **Social** — лента, посттар, интеракциялар.
- **Dating** — профильдер, сүзгілер, матчинг жұмысы.
- **Admin** — модерация, аналитика API.
- **Notification** — push, email (бөлек worker).

## 2. Технология стегі

| Қабат | Технология | Неге |
|-------|------------|------|
| Web | React 18 + TypeScript + Vite | Экожүйе, жылдам сборка, типтер |
| Mobile (келесі фаза) | React Native немесе Flutter | Бір команда, кроссплатформа |
| API | Node.js + Express (MVP) / кейін NestJS | Async I/O, WebSocket интеграциясы |
| Real-time | Socket.IO немесе native WS + Redis adapter | Горизонтальды масштаб |
| DB | PostgreSQL | ACID, JSONB, толықтай текст іздеу (кейін Elasticsearch) |
| Кэш / сессия | Redis | Rate limit, pub/sub, Socket.IO adapter |
| Файлдар / видео | S3-сүйінді объекті сақтау (MinIO / AWS S3) + CDN | Масштабталатын медиа |
| Контейнерлер | Docker + Kubernetes (прод) | Ортақ орта, автомасштаб |

## 3. Деректер базасы

**Негізгі:** **PostgreSQL** — реляциялық модель (пайдаланушылар, курстар, жазбалар), транзакциялар, индекстер.

**Қосымша:**
- **Redis** — кэш, сессия, real-time presence, Socket.IO кластері арасында хабарлама маршрутизациясы.
- **Elasticsearch/OpenSearch** (кейін) — лента, іздеу, модерация кезіндегі сүзгілеу.

**Хабарламалар:** серверде **тек шифрланған blob** (plaintext жоқ) — бұл талапқа сәйкес «бұлтта оқылатын мәтін жоқ» моделі (Signal-подобно: сақтау дегеніміз — шифрланған пакет).

---

## 4. Real-time (чат, хабарландырулар)

1. **WebSocket** (Socket.IO): клиент JWT handshake арқылы қосылады.
2. **Хабарлама:** клиент E2EE шифрлайды → сервер тек `ciphertext` + метадеректерді сақтайды → алушыға ретрансляция.
3. **Push:** FCM/APNs үшін **Notification** сервісі; триггер — жаңа шифрланған хабарлама немесе әлеуметтік оқиға.
4. **Кластер:** Redis adapter — бірнеше WS нодалары бір бөлмедегі қосылымдарды көрсетеді.

---

## 5. Миллиондарға масштабтау

- **Көлденең масштаб:** stateless API + Redis + PostgreSQL read replicas.
- **Медиа:** CDN, HLS/DASH сегменттер, signed URL.
- **Хабарлама:** WS нодаларының пулы, Redis pub/sub.
- **Лента:** кэшталған таспалар (Redis), кейін — fan-out on write / ActivityPub-стиль инженериялық шешім.
- **Бақылау:** OpenTelemetry, Prometheus, Grafana.
- **Шектеу:** API gateway rate limit, WAF.

```

            [Web/Mobile]   |    [Admin Web]

              |     Модульдік монолит    |

              +------------+------------+
                           |
        +------------------+------------------+
        |                  |                  |

        |                  |                  |
        |            [Socket.IO nodes]        |
        |                  |                  |
        +------------------+------------------+
                           |
                    [Push FCM/APNs]
                           |
                    [Analytics pipeline]
                    (кейін: Kafka → DW)
```

## 7. MVP шектеулері

Бұл репозиторийдегі MVP: SQLite + бір Node процесі + жеңілдетілген E2EE (клиент шифрланған payload жібереді; сервер plaintext көрмейді). Продакшнда PostgreSQL + Redis + толық Signal Protocol клиенті қажет.
