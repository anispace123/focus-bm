# FOCUS.bm — қауіпсіздік моделі

## 1. End-to-End шифрлау (Signal-стиль тұжырымдама)

- **Double Ratchet** + **X3DH** кілм алмасу — **клиенттік** кітапханаларда (мысалы, libsignal-ге ұқсас портация немесе `@privacyresearch/libsignal-protocol-typescript`).
- Сервер **жария кілттерді** (pre-keys, signed pre-keys) сақтай алады — бұл Signal-дағы сияқты; **хабарлама мәтіні серверге ашық түрде келмейді**.
- Серверде сақталатын нәрсе: **шифрланған blob**, жіберуші/алушы идентификаторлары, уақыт белгісі, тіркелгі идентификаторы (metadata заңды талаптарға бағынуы мүмкін).

**«Бұлтта хабарламалар жоқ» нақты мағынасы:** бұлтта **оқылатын мәтін жоқ**; техникалық тұрғыдан ретрансляция үшін шифрланған пакеттер уақытша сақталуы мүмкін (офлайн алушы үшін), бірақ сервер кілті жоқ.

## 2. Zero-knowledge элементтері

- **Құпия сөздер:** серверде тек **Argon2id** хэші; құпия сөзді білмейді.
- **Дейтинг сүзгілері:** мүмкіндігінше **клиентте** есептеледі немесе гомоморфты/жинақталған схемалар — MVP-де минималды метадерек + ерікті бөлісу.
- **Оқыту прогресі:** агрегаттар ғана аналитикаға; жеке бөлшектерді сату жоқ (саясат + техника).

## 3. Аутентификация

- **Access JWT** (қысқа мерзім) + **Refresh token** (ротация, httpOnly cookie ұсынылады веб үшін).
- **OAuth2** (Google/Apple): кейінгі фаза; state + PKCE.

## 4. MITM қорғанысы

- **TLS 1.2+** барлық сыртқы байланыстарда; **HSTS**; **certificate pinning** мобильді қосымшада.
- Public key pinning API үшін ұзақ мерзімді стратегия (ескі клиенттерді бұзбау).

## 5. SQL инъекциялары

- **ORM (Prisma)** — параметрленген сұраулар; raw SQL болса — тек байндинг.

## 6. XSS

- React әдепкі экрандау; **CSP** (Content-Security-Policy); dangerouslySetInnerHTML-ден аулақ болу; санитизация rich text болса — DOMPurify.

## 7. Басқа

- **Rate limiting** (Redis): логин, API, WS.
- **Helmet** (HTTP қауіпсіздік бастықтары).
- **CORS** тек сенімді origin-дер.
- **bcrypt/Argon2:** MVP-де **argon2** пароль үшін.

## 8. MVP кодтағы нақты іске асыру

- Парольдер: Argon2.
- JWT: HS256 (dev) / RS256 (prod ұсыныс).
- Хабарлама денесі: клиент `ciphertext` (base64) жібереді — сервер тек сақтайды және ретрансляциялайды; **дешифрлау серверде жоқ**.
