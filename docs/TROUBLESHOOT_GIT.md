# Репозиторийге папкалар «қосылмайды» — шешім

## 0. «git не является внутренней или внешней командой» (Windows)

Git орнатылмаған немесе терминал ескі PATH-ты көріп тұр.

**Орнату:**

```powershell
winget install Git.Git --accept-source-agreements
```

немесе [git-scm.com/download/win](https://git-scm.com/download/win) — орнатқышты іске қосыңыз, **«Add Git to PATH»** опциясын қосыңыз.

**Содан кейін:** PowerShell / CMD / Cursor терминалын **толығымен жабып қайта ашыңыз** (жаңа сессия PATH жаңартады).

Тексеру:

```bash
git --version
```

Егер әлі табылмаса, Git әдетте: `C:\Program Files\Git\bin\git.exe` — осы жолды PATH-қа қолмен қосуға болады.

## 1. Негізгі себеп: Git бос қалтаны сақтамайды

Git тек **файлдарды** бақылайды. Қалтада **ешбір файл жоқ** болса, репозиторийде ол қалта **көрінбейді**. Бұл қате емес — дизайн бойынша солай.

**Не істеу керек:** сол қалтаға кем дегенде бір файл салыңыз (мысалы бос `.gitkeep`):

```text
my-folder/.gitkeep
```

Содан кейін:

```bash
git add my-folder/.gitkeep
git commit -m "Add folder"
```

---

## 2. Дұрыс қалтада тұрғаныңызды тексеріңіз

Репозиторий түбі **`FOCUS.bm`** қалтасы болуы керек (ішінде `backend`, `frontend`, `README.md` бар).

```bash
cd путь\к\FOCUS.bm
git status
```

Егер `fatal: not a git repository` шықса, алдымен:

```bash
git init
```

---

## 3. `node_modules` және `.gitignore`

`.gitignore` ішінде `node_modules/` бар — сондықтан **осы аттас қалта GitHub-қа кірмейді**. Бұл дұрыс.

Код қалталары (`backend/src`, `frontend/src` т.б.) **файлдарымен** қосылуы керек. Егер IDE-да «папка бос» сияқты көрінсе, алдымен:

```bash
cd backend
npm install
cd ..\frontend
npm install
```

Содан кейін репозиторий түбінде:

```bash
cd ..\..
git add .
git status
```

Мұнда `backend/src/...`, `frontend/src/...` файлдары **көрінуі** керек; `node_modules` көрінбеуі керек.

---

## 4. Бір реттік толық қосу

`FOCUS.bm` түбінде:

```bash
git add -A
git status
git commit -m "FOCUS.bm: барлық файлдар"
```

Егер үлкен біреу бұғаттаса, `git status` қандай файлдардың «қосылғанын» көрсетеді.

---

## 5. GitHub веб-сайтында жүктеу

GitHub.com **бос қалтаны** жеке жүктеп салуға мүмкіндік бермейді — файл керек. Үлкен жобаны **Git push** арқылы жіберу керек (немесе GitHub Desktop).

---

## 6. Кіріктірілген (nested) `.git` қалтасы

Егер `FOCUS.bm` ішінде немесе ішкі қалтада **қосымша `.git`** болса, Git оны бөлек репо ретінде көреді — ішкі файлдар бас репоға түспейді.

**Тексеру:** `FOCUS.bm` және ішкі қалталарда `.git` бар ма.

**Шешім:** ішкі `.git` қалтасын жойыңыз (немесе submodule ретінде баптаңыз — күрделірек).

---

## 7. Қысқа чек-лист

| Мәселе | Шешім |
|--------|--------|
| Бос қалта көрінбейді | Ішіне файл (мысалы `.gitkeep`) салыңыз |
| `git add` ештеңе қоспайды | `FOCUS.bm` түбінде екеніңізді тексеріңіз, `git init` |
| Тек `node_modules` жоқ | Қалыпты — ол `.gitignore`-да |
| Идеда папка «сұр» | `git add .` кейін қайта ашыңыз немесе commit жасаңыз |
